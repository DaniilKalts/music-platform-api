# 🎵 Music Platform API

Backend-сервис музыкальной платформы (по типу Spotify / Яндекс Музыки) — REST API. Пользователи регистрируются, ищут и «прослушивают» треки, создают плейлисты и добавляют избранное в рамках одного из двух типов подписки (**FREE** / **PREMIUM**). Каталог наполняют только администраторы. 

Цель проекта — закрепить ключевые навыки backend на Go: REST API на чистой архитектуре, PostgreSQL с SQL-миграциями, JWT-аутентификацию, middleware, контроль доступа по ролям и подпискам, кэширование через Redis, логирование, Swagger, Docker / Docker Compose и тесты.

---

## ✨ Возможности

- 🔐 **Аутентификация и авторизация**
    - Регистрация по email, имени пользователя и паролю
    - Вход с выдачей **access-токена** + **refresh-токена** (JWT)
    - Обновление access-токена по refresh-токену
    - Выход
    - Две роли: **USER** (по умолчанию) и **ADMIN**

- 👤 **Управление профилем**
    - Просмотр собственного профиля (id, email, имя пользователя, роль, тип подписки, дата создания)
    - Обновление собственного профиля

- 🎧 **Треки**
    - Список каталога с пагинацией
    - Получение одного трека по ID
    - Поиск по названию / исполнителю / жанру / альбому
    - **Прослушивание трека:** проверка существования → запись в историю → возврат данных трека

- 📃 **Плейлисты** _(только свои)_
    - Создание, чтение, обновление, удаление плейлистов
    - Добавление трека в плейлист
    - Удаление трека из плейлиста
    - Для FREE — не более **3 плейлистов**

- ⭐ **Избранное**
    - Добавление трека в избранное
    - Получение списка избранного
    - Удаление трека из избранного
    - Для FREE — ограниченное количество треков

- 🕑 **История прослушиваний**
    - Просмотр истории: id трека, название, исполнитель и время прослушивания

- 💳 **Типы подписки**
    - **FREE:** безлимитные прослушивания и скачивание, но не более **3 плейлистов** и ограниченное избранное
    - **PREMIUM:** безлимитные плейлисты и избранное; расширенный функционал
    - Смена типа подписки (FREE ↔ PREMIUM) выполняется через admin-эндпоинт

- 🛠 **Администрирование**
    - Добавление / обновление / удаление треков (**только мягкое удаление** — никогда не hard delete)
    - Изменение типа подписки любого пользователя

- ⚙️ **Сквозные задачи**
    - **Middleware:** логирование HTTP-запросов, восстановление после panic, проверка JWT, проверка роли администратора, Request ID
    - **Лимиты подписки:** проверяются в сервисном слое
    - **Кэширование (Redis):** популярные треки, жанры, трек по ID, результаты поиска
    - **Структурированное логирование (zap):** запросы, ошибки, события авторизации, создание плейлистов, прослушивания, превышение лимитов, действия администратора
    - **Единообразные ошибки:** понятные JSON-ошибки (`{ "error": "track not found" }`) с корректными HTTP-кодами
    - **Swagger**-документация с авторизацией через Bearer Token

## 🛠 Технологический стек

| Категория | Технология |
|---|---|
| **Язык** | Go |
| **Архитектура** | Чистая: Handler → Service → Repository |
| **БД** | PostgreSQL (SQL-миграции) |
| **Кэш** | Redis |
| **Хранилище аудио** | RustFS (S3-совместимое): файлы лежат в бакете, `tracks.file_url` ссылается на объект |
| **Аутентификация** | JWT (access + refresh токены) |
| **Логирование** | zap (структурированное) |
| **Документация** | Swagger (Bearer Token) |
| **Контейнеризация** | Docker + Docker Compose (backend + postgres + redis + rustfs) |
